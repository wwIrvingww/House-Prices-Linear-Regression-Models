Probando con diferentes profundidades

# Definir configuraciones de profundidad
depth_settings <- list(
  modelo1 = list(maxdepth = 4),
  modelo2 = list(maxdepth = 6),
  modelo3 = list(maxdepth = 8)
)

# Crear lista para almacenar los modelos
resultados <- list()

# Entrenar los modelos con diferentes profundidades
for (nombre in names(depth_settings)) {
  set.seed(123)
  modelo <- train(
    consumoCat ~ ., 
    data = train_data, 
    method = "rpart", 
    trControl = trainControl(method = "cv", number = 5),
    tuneLength = 10,  # Ajusta automticamente cp probando 10 valores diferentes
    control = rpart.control(maxdepth = depth_settings[[nombre]]$maxdepth)
  )
  resultados[[nombre]] <- modelo
}

accuracy_results <- data.frame(
  Modelo = character(),
  Profundidad = numeric(),
  Accuracy = numeric(),
  stringsAsFactors = FALSE
)

for (nombre in names(resultados)) {
  profundidad <- depth_settings[[nombre]]$maxdepth
  accuracy <- max(resultados[[nombre]]$results$Accuracy)

  accuracy_results <- rbind(accuracy_results, data.frame(
    Modelo = nombre,
    Profundidad = profundidad,
    Accuracy = accuracy
  ))

  print(paste("Mostrando el rbol:", nombre, "- Profundidad:", profundidad))

  rpart.plot(resultados[[nombre]]$finalModel, main = paste("rbol con", nombre, "\nProfundidad:", profundidad))
}

Como se puede observar al final este fue el mejor resultado de todos los modelos que fueron entrenados, superando incluso al algoritmo de regresin lineal y las clasificaciones realizadas anteriormente. Es incluso mejor que el random forest de regresion. Logrando un Accuracy de 83%, este algoritmo es mucho ms pesado que los dems debido a que aparte de hacer tuning para la mejora de hiperparmetros, se estn realizando una gran cantidad de rboles para poder determinar variables ms importantes. Al tener muchos arboles (500 en este caso de prueba), lo que hace esto es haer muchas predicciones dentro del entrenamiento con los distintos arboles. Los rboles tienen bastante variabilidad, ya que el muestro involucra distintos procesos aleatorios, lo cul nos garantiza una mayor generalizacin y menos overfitting (el cul fue el problema que tuvimos con la regresin lineal). Adems, en este caso la clasificacin ayuda bastante a que el modelo pueda predecir sobre una variable discreta (categora de la casa) en un lugar de una variable continua (el precio). Ya que se tienen definidos rangos es que el mdelo busca la mayor coincidencia entre las predicciones de los distintos rboles. Esta discretizacin ayuda mucho a mejroar el desempeo y nos permiti un realizar un modelo con alto accuracy tomando en cuenta las variables ms relevantes. Como se puede ver en la grfica, las variables ms importante coinciden con las que se analizaron tanto la regresin lineal como en el arbol de clasificacin. Por lo que podemos concluir que el modelo entrenado es bastante preciso y generalizado.Otro indicador adems de l accuracy es el kappa. Este result ser mucho mayor a 0.50 (0.75 al final), lo cul indica que el mdelo no solo predice aleatoriamente y que de hecho es mejor que una clasificacin aleatoria.
